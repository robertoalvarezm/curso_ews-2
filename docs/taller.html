B.1 Flujo local (no se ejecuta aquí)

# Python (local) con ewstools
import pandas as pd, ewstools
serie = pd.read_csv("paleoclima.csv")["valor"]      # ajusta a tu archivo
ts = ewstools.TimeSeries(data=serie)
ts.detrend(method="Lowess", span=0.2)
ts.compute_var(rolling_window=0.5)
ts.compute_auto(rolling_window=0.5, lag=1)
ts.compute_ktau()
ts.make_plotly().show()
# R (local) con EWSmethods
library(EWSmethods)
df <- read.csv("paleoclima.csv")          # columnas: tiempo, valor
res <- uniEWS(data = df[, c("tiempo","valor")],
              metrics = c("SD","ar1","skew"),
              method = "rolling", winsize = 50)
plot(res)

3 Cierre crítico

Las EWS son una herramienta poderosa pero no infalible (Boettiger y Hastings 2012; Dakos et al. 2012):

  • Falsos positivos: varianza/autocorrelación pueden subir por ruido externo o cambios de muestreo sin que haya pérdida de resiliencia.
  • Falsos negativos: transiciones inducidas por ruido o por choques abruptos del forzamiento pueden no dejar firma anticipatoria.
  • Dependencia de elecciones: detrending, tamaño de ventana y frecuencia de muestreo cambian el resultado; siempre reporta sensibilidad y usa surrogates para la significancia.
  • El tipo importa: la firma clásica corresponde a bifurcaciones tipo fold; otras transiciones requieren métodos distintos (de ahí el atractivo de los enfoques de ML del capítulo 7).
